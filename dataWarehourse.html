Project 2 : Data Warehousing

Introduction

This project was aimed to design a relational database for a shipping firm based on a shipment invoice. Dimensional modelling method was used to create a star schema. Fig 2.1 below shows the start schema of the shipping company (FK means foreign key).

Software used: Draw.io , SQL Developer

Explanation

The business process behind designing this dimensional model is to acknowledge the most granular business questions on the shipment invoice subject to shipment, customer, product, warehouse, deals, payment method, etc. The shipment company serves limited number of customers while all customers are registered to the company and may ship any items to any customer.

A few key aspects when designing this database are:

  1. Fact table has been normalized to ensure single generic fact along with the dimension that identifies the type of fact.
  2. Attributes like shipment mode, product, deal, contract and warehouse are treated in a separate dinension.
  3. Total billed amount is computable from tax, total billed amount and charges.
  4. Facts are available at the most ganular level.
  5. Snowflaking customer dimension into an outrigger has been done for cluster of low cardinalities to browse and track changes.

This database is thus designed to be queried using SQL, or data access tools. While making this database, I crucially thought that the database should adapt to businesses' dynamics changes, limitations, query processing speed and accuracy.
